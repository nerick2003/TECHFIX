OVERVIEW:
---------
This guide contains 15 journal entries to manually input into your TechFix 
accounting system. These entries include:
  • 10 Regular Transactions (Entries 1-10)
  • 3 Adjusting Entries (Entries 11-13)
  • 2 Closing Entries (Entries 14-15) - Generated automatically by the system

IMPORTANT NOTES:
----------------
1. Replace YYYY with the current year (e.g., 2025)
2. All amounts are in your local currency (PHP/Peso)
3. Ensure accounts exist in your Chart of Accounts before entering
4. Entries must be entered in chronological order for best results
5. Closing entries (14-15) are generated automatically - do NOT enter manually

PHASE 3: CLOSING ENTRIES (Entries 14-15)
═══════════════════════════════════════════════════════════════════════════════

⚠️ DO NOT ENTER THESE MANUALLY!

Closing entries are generated automatically by the system. To create them:

1. Go to the "Financial Statements" tab
2. Ensure all adjusting entries are completed
3. Click "Generate Closing Entries" or use the closing entries feature
4. The system will automatically create closing entries on YYYY-02-01

The closing entries will:
  • Close all revenue accounts to Income Summary
  • Close all expense accounts to Income Summary
  • Close Income Summary to Owner's Capital
  • Close Owner's Drawings to Owner's Capital (if applicable)

═══════════════════════════════════════════════════════════════════════════════
STEP-BY-STEP INSTRUCTIONS
═══════════════════════════════════════════════════════════════════════════════

HOW TO INPUT EACH ENTRY:

1. Open TechFix Accounting System
2. Navigate to the "Transactions" tab
3. For each entry above, follow these steps:

   a) Enter the Date:
      • Format: YYYY-MM-DD (e.g., 2025-01-01)
      • Use the date picker or type manually

   b) Enter the Description:
      • Copy exactly as shown in the entry above

   c) Select Source Type (Optional):
      • Choose from dropdown: Receipt, Invoice, Bank, Payroll, Adjust, etc.

   d) Enter Document Reference (Optional):
      • Leave blank or enter a reference number

   e) Enter External Reference (Optional):
      • Leave blank or enter an external reference

   f) For ADJUSTING ENTRIES (11-13):
      • CHECK the "Adjusting Entry" checkbox!

   g) Select Debit Account:
      • Click the Debit Account dropdown
      • Select the account name exactly as shown
      • Account names must match exactly (case-sensitive)

   h) Enter Debit Amount:
      • Enter the amount as shown (e.g., 150000 or 150000.00)
      • You can use commas: 150,000.00

   i) Select Credit Account:
      • Click the Credit Account dropdown
      • Select the account name exactly as shown

   j) Enter Credit Amount:
      • Enter the amount as shown
      • The system will auto-fill if you only enter debit amount

   k) Verify Balance:
      • Ensure Debit Amount = Credit Amount
      • System will show error if not balanced

   l) Click "Record & Post":
      • This saves and posts the entry immediately
      • You'll see a confirmation message with entry ID

4. Repeat for all 13 manual entries (entries 14-15 are automatic)

5. After entering all entries, generate closing entries:
   • Go to Financial Statements tab
   • Click "Generate Closing Entries" or use the closing feature
   • System will create closing entries automatically

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

PROBLEM: "Could not resolve account" error
SOLUTION: 
  • Check that account name matches exactly (case-sensitive)
  • Ensure account exists in Chart of Accounts
  • Use dropdown to select account, don't type manually

PROBLEM: "Debits must equal credits" error
SOLUTION:
  • Verify amounts are entered correctly
  • Check for typos in amounts
  • Ensure both debit and credit amounts match

PROBLEM: Financial statements show imbalance
SOLUTION:
  • Verify all entries are posted (not drafts)
  • Check that adjusting entries are marked as "Adjusting Entry"
  • Ensure closing entries have been generated
  • Verify all account names match exactly

PROBLEM: Accounts not appearing in dropdown
SOLUTION:
  • Go to Chart of Accounts and verify accounts exist
  • Refresh the Transactions tab
  • Restart the application if needed
